How to Actually Do It

Two distinct paths for working with AI.

The Architect

Best for: Blank-canvas ideas & new projects

  • Plan First: Have a conversation with Gemini before asking for any code.
  • Map it out: Discuss architecture, feasibility, and best practices.
  • Execute: Once the blueprint is nailed down, go into the trenches and build it piece by piece.

The Validator

Best for: Iterations & delegating features

  • Delegate: Give the AI a small, highly specific task or upgrade.
  • Let it work: Allow it to write the initial logic and structure.
  • Review: Your job shifts entirely to reviewing, testing, and directing tweaks.

Reality Check

The Catch: What AI Leaves Out

Don't build a tool that accidentally leaks your customer database.

The "Eager Intern" Problem

AI wants to give you working code as fast as possible. This means it will happily cut corners on security and ignore edge cases if you don't explicitly tell it not to.

Hardcoded Secrets

AI will often put your API keys or passwords right in plain text in the script. Always use Environment Variables (`.env` files) to keep secrets safe.

Security Vulnerabilities

It won't always account for things like SQL injections or cross-site scripting (XSS) unless you prompt it to be secure by design.

The Director's QA Prompt

"Review this code for security vulnerabilities, edge cases, and best practices before we finalize."
